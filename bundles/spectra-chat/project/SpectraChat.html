Great project scope. For a production-grade React dashboard with real-time data, I'd recommend this layered architecture:

1. State layer — Use Zustand for global UI state and React Query for server state with WebSocket subscriptions.
2. Layout — CSS Grid for the shell, a collapsible sidebar using a context provider.
3. Charts — Recharts is the pragmatic choice; Tremor gives you pre-built dashboard components if you want to move faster.

For FastAPI + React WebSocket integration, the cleanest pattern is a custom hook that wraps a useRef socket instance with React Query's setQueryData to push updates directly into the cache:
tsx 
export function useRealtime(channel: string) {
  const qc = useQueryClient()
  const ws = useRef<WebSocket>()

  useEffect(() => {
    ws.current = new WebSocket(
      `wss://api.example.com/ws/${channel}`
    )
    ws.current.onmessage = (e) => {
      const data = JSON.parse(e.data)
      qc.setQueryData(['metrics', channel], data)
    }
    return () => ws.current?.close()
  }, [channel])
}
